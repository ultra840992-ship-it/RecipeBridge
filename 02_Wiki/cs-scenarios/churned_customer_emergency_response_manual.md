# 이탈 고객 긴급 우회 대응 매뉴얼: 회원가입 인증 코드 대기 구간 이탈 집중 관리

## 1. 개요 및 목표

저희 서비스의 베타 가입 퍼널 분석 결과, 회원가입 시 '인증 코드 확인 대기' 구간에서 35%라는 높은 이탈률이 확인되었습니다. 이는 고객 여정 중 가장 큰 고통 지점으로 판단되며, 긴급한 우회 대응이 필요합니다. 본 매뉴얼은 해당 구간에서 이탈한 고객의 불편을 최소화하고, 가입 완료를 유도하며, 동시에 1:1 피드백을 수집하여 제품 개선에 기여하는 것을 목표로 합니다.

## 2. 문제 정의 및 원인 분석

*   **문제 지점**: 회원가입 시 '인증 코드 확인 대기' 단계
*   **이탈률**: 35% (GA4 1차 추적 결과, 2026-07-18)
*   **추정 원인**:
    *   API 비동기 가입 절차의 복잡성으로 인한 시스템 지연.
    *   인증 코드 전송 지연 또는 미수신으로 인한 고객 대기 시간 증가 및 답답함.
    *   명확하지 않은 대기 안내 메시지로 인한 고객 혼란.

## 3. 긴급 우회 대응 가이드라인 (CS 에이전트용)

이탈 고객과의 접점에서 다음과 같은 절차를 따라주십시오.

### 3.1 이탈 고객 식별 및 선제적 접근

1.  **대상 고객 리스트 확보**: GA4 퍼널 데이터 및 CS 시스템 내 회원가입 이탈 로그를 통해 '인증 코드 확인 대기' 구간에서 이탈한 고객 리스트를 주기적으로 확보합니다.
2.  **복귀 유도 메일 발송**: 이전에 발행된 '이탈 유저 복귀 리드메일 티켓' 명세서에 따라, 고객의 이탈 시점으로부터 30분 이내에 개인화된 복귀 유도 이메일 또는 메시지를 발송하여 재가입을 독려합니다.
    *   **메일 문구 예시**: "안녕하세요, 사장님. 회원가입 중 혹시 불편을 겪으셨을까요? [서비스명] 가입 여정을 다시 시작하시면 성공적으로 완료하실 수 있도록 저희가 적극적으로 도와드리겠습니다."

### 3.2 1:1 고객 접점 시 대응 스크립트 및 피드백 수집

고객이 직접 문의하거나, 선제적 접근 후 회신이 온 경우, 다음 스크립트를 활용하여 응대하고 피드백을 수집합니다.

1.  **공감 및 사과**:
    *   "사장님, 회원가입 중 불편을 드려 정말 죄송합니다. 혹시 인증 코드 입력 단계에서 어려움을 겪으셨을까요?"
    *   "현재 해당 구간에서 일부 고객분들이 비슷한 불편을 겪고 계시며, 저희가 이 문제를 빠르게 인지하고 개선하기 위해 노력하고 있습니다."
2.  **긴급 우회 방안 제시 (필요시)**:
    *   "지금 바로 가입을 완료하실 수 있도록, 저희가 수기 인증 절차를 진행해 드리거나, 다른 이메일 주소로 재시도를 도와드릴 수 있습니다. 어떻게 도와드릴까요?"
    *   (수기 인증 절차가 불가능할 경우) "현재 시스템 개선 중이라 다소 시간이 소요될 수 있습니다. 불편하시겠지만 잠시 후 재시도해주시거나, 다른 이메일 주소로 시도해보시면 원활하게 진행될 수 있습니다."
3.  **1:1 피드백 적극 수집**:
    *   "가입 완료 후, 해당 과정에서 어떤 점이 가장 불편하셨는지 짧게 피드백 주시면 서비스 개선에 큰 도움이 됩니다. 사장님의 소중한 의견을 제품팀에 전달하여 더 나은 서비스를 만드는 데 반영하겠습니다."
    *   **수집 항목**:
        *   인증 코드 수신까지 걸린 시간
        *   인증 코드 미수신 여부
        *   대기 중 느꼈던 감정 (답답함, 불안감 등)
        *   개선되었으면 하는 점

### 3.3 피드백 기록 및 연계

1.  **CS 티켓 상세 기록**: 고객과의 모든 상호작용 내용 및 수집된 1:1 피드백은 CS 티켓에 상세히 기록합니다.
2.  **서비스 개선 백로그화**: 기록된 피드백은 `02_Wiki/decisions/churn_customer_feedback_YYYYMMDD.md` 형태로 자산화하여 서비스 개선 백로그로 활용합니다.
3.  **내부 피드백 루프 연계**: 수집된 1:1 피드백은 마스터(Aegis)를 통해 기획(Nova)에 즉시 연계하여 다음 스프린트 제품 개선에 반영되도록 조치합니다.

## 4. 장기 개선 계획 (Bitz 연계)

근본적인 문제 해결을 위해 Bitz(개발팀)에게 API 비동기 가입 절차 간소화를 요청하여 인증 코드 전송 및 확인 과정을 더욱 빠르고 안정적으로 개선할 예정입니다. 이탈 고객 긴급 우회 대응 매뉴얼은 해당 개선이 완료될 때까지 임시적으로 운영됩니다.

## 1. AI 가상 조직도 및 역할 정의 (R&R)

각 가상 에이전트는 독립적인 프롬프트 컨텍스트와 할당된 도구 권한(MCP, Web API)을 가지고 임무를 수행합니다.

```
                  ┌───────────────────────┐
                  │      마스터 에이전트    │ (전체 통제 & 파일 검수)
                  └───────────┬───────────┘
                              │
         ┌────────────┬───────┴────┬────────────┬────────────┐
         ▼            ▼            ▼            ▼            ▼
     ┌───────┐    ┌───────┐    ┌───────┐    ┌───────┐    ┌───────┐
     │ 기획  │    │디자인 │    │ 개발  │    │마케팅 │    │  CS   │
     │ 에러  │    │ 벤치  │    │빌드 & │    │ SEO & │    │피드백 │
     │ 레시피│    │ 마킹  │    │ 디버깅│    │카드뉴스│    │ 수집  │
     └───────┘    └───────┘    └───────┘    └───────┘    └───────┘
```

### 👤 1. 마스터 에이전트 (Master Coordinator)
* **목표**: 5인 부서의 업무 조율, 과제 할당 및 최종 아웃풋의 품질 검수(거짓말 추적).
* **사원증 및 장비 권한**: `C:\Users\ultra\AI 1인 기업화` 전체 폴더 읽기/쓰기, Git 커밋 권한.

### 👤 2. 기획 에이전트 (Service Planner)
* **목표**: 스타트업 의뢰 과제에서 필요한 UI 뼈대 정의 및 레시피 빌더용 '프롬프트 밀키트' 기획.
* **사원증 및 장비 권한**: 노션 API 커넥터 (과제 정의서 DB 연동).

### 👤 3. 디자인 에이전트 (UI/UX Designer)
* **목표**: 크로스오버 벤치마킹을 기반으로 한 웹사이트 색상, 폰트, 버튼 스타일 등 테마 요소 가이드 도출.
* **사원증 및 장비 권한**: 피그마 API 커넥터 (와이어프레임 데이터 송수신).

### 👤 4. 개발 에이전트 (Lead Developer)
* **목표**: 클로드 코드를 호출하여 레시피대로 10분 프로토타입 구현 및 소셜 로그인 등 기능 보강 디버깅.
* **사원증 및 장비 권한**: 로컬 개발 서버 실행 권한(`npm run dev`), npm/npx 패키지 설치 권한.

### 👤 5. 마케팅 에이전트 (Growth Marketer)
* **목표**: 검색 노출 극대화(SEO) 및 매일 SNS에 발행할 카드뉴스 콘텐츠 자동 기획/제작.
* **사원증 및 장비 권한**: 구글 서치 콘솔 API, 인스타그램/유튜브 API 연동.

### 👤 6. CS 에이전트 (Customer Success)
* **목표**: 플랫폼 이용자(쌩신입 및 스타트업)들의 이탈 구간 모니터링 및 실시간 피드백 수집 요약.
* **사원증 및 장비 권한**: 구글 애널리틱스(GA4) API 커넥터.

## 2. 회사 매뉴얼 (`recipebridge_manual.md`) 핵심 규칙

에이전트들이 제멋대로 행동하지 않고 협업하도록 하기 위해 최상위 루트에 매뉴얼 파일을 배치하고 에이전트 실행 시 필수 임포트합니다.

1. **상호 검증 협업**: 기획 에이전트의 기획서가 완성되어 마스터의 승인을 받기 전까지는 개발 에이전트는 코딩을 시작하지 않는다.
2. **독립성 유지**: 다른 팀의 작업 파일을 직접 수정할 때는 반드시 마스터 에이전트에게 사전 승인을 구한다.
3. **토큰 최적화**: 개발 에이전트는 코드를 매번 새로 쓰지 않고, 디자인 에이전트가 정의한 UI 컴포넌트 레시피 팩을 부분 교체(Replace) 방식으로 덧붙인다.

## 3. 거짓말(할루시네이션) 방지를 위한 마스터 검수 규칙 2줄

AI 직원이 완료되지 않은 작업에 대해 "다 끝났습니다"라고 습관적으로 보고하는 것을 막기 위해 마스터 에이전트 프롬프트에 하단 **2줄의 하드코딩 검수 룰**을 주입합니다.

> ⚠️ **마스터 검수 강제 규칙**
> 1. 하위 부서(기획, 개발 등)가 작업을 완료했다고 보고하면, **최종 보고를 하기 전에 마스터 에이전트는 해당 결과물 파일(코드 또는 문서)을 직접 읽기 도구(view_file 등)로 열어서 실제 반영 여부와 증거를 명확하게 대조·확인**해야 한다.
> 2. 직접 눈으로 확인하지 못한 영역이 단 하나라도 있다면, 추정해서 "완료되었다"고 보고하지 말고 **"확인하지 못해 완료 여부를 보증할 수 없다"고 사용자에게 솔직하게 보고**해야 한다.

## 4. 24시간 자동화 및 수익화 시나리오

`/schedule` 크론 설정을 통해 인간 개입 없이 작동하는 일일 루틴을 셋업합니다.

```
[AM 09:00] 크론 구동 
    │
    ├──> 1. [CS팀] 구글 애널리틱스 분석 및 구직자 이탈 데이터 리포트 작성
    ├──> 2. [마케팅팀] 구글 서치 콘솔 노출 분석 및 유입 급감 키워드 추출
    │
    ▼
[AM 10:00] 회의 및 개선 실행
    │
    ├──> 3. [기획팀] 유입 급감 키워드를 타겟팅할 신규 프로젝트 과제 기획
    ├──> 4. [개발팀] 사이트 내 SEO 메타 태그 및 사이트맵 자동 갱신
    │
    ▼
[PM 02:00] 마케팅 배포
    │
    └──> 5. [마케팅팀] AI 이미지 제너레이터를 이용한 카드뉴스 이미지 생성 및 SNS 자동 업로드
    │
    ▼
[PM 06:00] 마스터 최종 검수
    │
    └──> 6. [마스터] 하루 동안 변경된 파일 목록 및 오류 로그 점검 후 요약 메일 발송
```

### 크론 명령어 예시 (CLI)
```bash
# 매일 오전 9시에 RecipeBridge 일일 마케팅 및 SEO 최적화 체인을 가동하는 크론 세팅
agy schedule --cron "0 9 * * *" --prompt "@GrowthMarketer @Master SEO 노출 상태를 점검하고, 이탈률 극복을 위한 신규 AI 레시피 및 과제를 기획/개발해줘."
```
*(주의: 로컬 PC 기반 에이전트 환경의 경우 데스크톱 전원이 켜져 있어야 하므로, 향후 상시 자동화를 위해 클라우드 VPS(가상 전용 서버) 환경으로 빌드를 이관할 예정입니다.)*

# 다국어(영어/일어) 대응 반응형 대시보드 UI/UX 디자인 가이드

## 1. 개요 (Overview)
본 문서는 프리미엄 라이트 테마(샌드 크림/샴페인 골드) 및 서리 낀 유리(Frosted Glassmorphism) 효과를 극대화한 대시보드 UI/UX가 다양한 디바이스(모바일, 태블릿, 데스크톱)와 다국어(영어, 일본어) 환경에서 일관되고 아름다운 사용자 경험을 제공할 수 있도록 하는 디자인 가이드라인을 제시합니다.

## 2. 핵심 디자인 원칙 (Core Design Principles)
-   **프리미엄 & 고급스러움**: 샌드 크림, 샴페인 골드 톤과 글래스모피즘 효과를 모든 환경에서 일관되게 유지하여 최고급 감성을 전달합니다.
-   **직관성 & 사용성**: 언어와 화면 크기에 관계없이 사용자가 쉽게 정보를 인지하고 상호작용할 수 있도록 설계합니다.
-   **반응성**: 모든 컴포넌트와 레이아웃은 다양한 화면 크기에 유연하게 대응하여 최적의 가독성과 조작성을 제공합니다.
-   **다국어 지원**: 텍스트 길이 변화, 폰트 특성 등을 고려하여 언어별 최적화된 UI를 제공합니다.

## 3. 디자인 토큰 (Design Tokens)

### 3.1. 색상 (Colors)
기존에 배포된 프리미엄 라이트 테마의 색상 팔레트를 다국어 및 반응형 환경에서도 동일하게 적용합니다.
-   **Primary**: #F5F5DC (Sand Cream)
-   **Accent**: #B8A774 (Champagne Gold)
-   **Background**: #FCFCF7 (Soft Off-White)
-   **Frosted Glass**: `rgba(255, 255, 255, 0.3)` with `backdrop-filter: blur(10px);`
-   (자세한 팔레트는 `02_Wiki/design/brand_guidelines.md` 참조)

### 3.2. 타이포그래피 (Typography)
가독성과 심미성을 모두 만족시키는 폰트 스택을 정의합니다.
-   **영어 (English)**:
    -   **Primary Font**: 'Pretendard', 'Roboto', 'Helvetica Neue', sans-serif
    -   **Fallback Font**: system-ui
-   **일본어 (Japanese)**:
    -   **Primary Font**: 'Noto Sans JP', 'Pretendard', sans-serif
    -   **Fallback Font**: system-ui, Yu Gothic, Meiryo
-   **폰트 사이즈 & 라인 하이트**:
    -   반응형 브레이크포인트에 따라 폰트 사이즈를 동적으로 조절합니다.
    -   모바일 환경에서는 최소 14px 이상을 유지하여 가독성을 확보합니다.
    -   라인 하이트는 폰트 사이즈의 1.5배를 기준으로 설정하여 텍스트 가독성을 높입니다.
-   **폰트 웨이트**: 텍스트의 중요도에 따라 Regular(400), Medium(500), SemiBold(600)를 사용합니다.

### 3.3. 간격 (Spacing)
-   8pt 그리드 시스템을 기반으로 모든 간격(margin, padding)을 설정하여 일관성을 유지합니다.
-   반응형 환경에서는 주요 컴포넌트 간의 간격을 최소화하거나 조정하여 공간 효율성을 높입니다.

### 3.4. 아이코노그래피 (Iconography)
-   모든 아이콘은 SVG 형태로 사용하며, 색상은 테마에 맞춰 변경 가능하도록 합니다.
-   문화적 중립성을 고려하여 특정 문화권에만 이해되는 아이콘은 지양합니다.

## 4. 반응형 디자인 가이드 (Responsive Design Guide)

### 4.1. 브레이크포인트 (Breakpoints)
-   **Mobile**: ≤ 767px
-   **Tablet**: 768px ~ 1023px
-   **Desktop**: ≥ 1024px

### 4.2. 레이아웃 (Layouts)
-   **Fluid Grid System**: Flexbox 또는 CSS Grid를 활용하여 유연한 그리드 레이아웃을 구현합니다.
-   **Stacking & Wrapping**: 모바일 환경에서는 컴포넌트를 수직으로 쌓거나(Stacking), 콘텐츠가 넘칠 경우 자동으로 줄 바꿈(Wrapping) 되도록 설계합니다.
-   **Primary Navigation**:
    -   데스크톱: 좌측 사이드바 또는 상단바 형태로 유지.
    -   모바일: 햄버거 메뉴 또는 하단 탭 내비게이션으로 전환.

### 4.3. 컴포넌트 스케일링 (Component Scaling)
-   **버튼**: 모바일에서는 터치 영역을 고려하여 최소 44x44px 이상으로 유지하며, 텍스트 길이가 길어질 경우 유동적으로 넓이를 조절합니다.
-   **카드/패널**: 화면 크기에 따라 너비를 유동적으로 조절하며, 모바일에서는 100% 너비로 확장될 수 있습니다.
-   **폼 요소**: 입력 필드는 모바일에서 100% 너비로 확장하여 사용성을 높입니다.

## 5. 다국어 UI/UX 가이드 (Multilingual UI/UX Guide)

### 5.1. 텍스트 길이 변화 대응 (Text Length Variation)
-   **가변 길이 UI 요소**: 버튼, 메뉴 항목, 카드 제목 등 텍스트가 들어가는 모든 UI 요소는 텍스트 길이에 따라 유연하게 너비가 조절되어야 합니다. (예: 일본어는 짧고, 영어는 길 수 있음)
-   **최대 텍스트 길이**: 버튼이나 탭 라벨 등 제한된 공간의 텍스트는 번역 시 최대 길이를 초과하지 않도록 가이드라인을 제시하고, 초과 시에는 말줄임표(...) 처리를 적용합니다.
    -   예시: 버튼 텍스트는 최대 15자(영문 기준)를 넘지 않도록 권고.
-   **레이아웃 유지**: 텍스트가 길어져도 주변 컴포넌트나 전체 레이아웃이 깨지지 않도록 충분한 여백을 확보합니다.

### 5.2. 폰트 선택 (Font Selection for Languages)
-   위 3.2 타이포그래피에서 정의된 각 언어별 폰트 스택을 사용하여 일관된 가독성과 프리미엄 감성을 유지합니다.

### 5.3. 날짜/시간/숫자 형식 (Date/Time/Number Formats)
-   각 언어 및 문화권에 맞는 날짜, 시간, 숫자 형식을 사용합니다.
    -   예시: 영어 (MM/DD/YYYY), 일본어 (YYYY年MM月DD日)
-   숫자 천 단위 구분자도 해당 문화권의 규칙을 따릅니다.

### 5.4. 방향성 (Directionality)
-   현재는 LTR(Left-to-Right) 언어(영어, 일본어)를 기본으로 합니다. 향후 RTL(Right-to-Left) 언어 지원 시에는 별도의 디자인 가이드라인을 수립합니다.

### 5.5. 번역 가이드라인 (Translation Guidelines)
-   전문 번역가를 통해 번역하며, UI/UX 문맥을 고려한 자연스러운 번역을 지향합니다.
-   기술 용어는 통일된 번역 원칙을 따릅니다.

## 6. 글래스모피즘 효과 (Frosted Glassmorphism Effect)
-   다국어 및 반응형 환경에서도 프리미엄 글래스모피즘 효과는 일관되게 유지되어야 합니다.
-   모바일 환경에서는 성능 저하가 발생하지 않도록 CSS 최적화에 유의합니다.

## 7. 에이전트 아바타 (Agent Avatars)
-   기존에 생성된 고급스러운 6인 에이전트 아바타는 다국어 환경에서도 변함없이 동일한 비주얼을 유지하여 브랜드 아이덴티티를 강화합니다.

# [실전 EP2] 토큰 소모 최소화! AI 레시피 조합으로 10분 만에 서비스 구현하기

> 💡 **핵심 인사이트**
> "좋은 예술가는 베끼고 위대한 예술가는 훔친다" - 스티브 잡스(피카소 인용)
> 무작정 프롬프트를 길게 짜서 토큰을 낭비하는 '무지성 개발' 대신, 이미 검증된 UI와 기능 재료를 **'레시피 팩'**으로 조합해 AI(클로드 코드 등)에게 건네주는 것이 비용과 시간을 아끼는 창업 전략이다.

---

## 1. 유료화된 도구의 한계 극복 (토큰 최적화 전략)

* **문제 상황:** 디자인 AI인 '피그마 메커(Figma Maker)'가 베타 기간을 끝내고 유료로 전환되면서 무작정 테스트하기엔 토큰 비용 부담이 커짐 [00:00:46].
* **해결책:** 디자인과 기능 요소를 미리 밀키트처럼 패키징한 **'레시피 빌더 도구'**를 활용하여 최소한의 AI 요청으로 최대의 퀄리티를 뽑아내는 방식으로 선회 [00:01:05].

---

## 2. 합법적 모방을 통한 서비스 구조 기획 (UI/UX 훔치기)

* **잘못된 모방:** 동종 업계 경쟁사의 UI를 그대로 복사해 상품화하면 부정경쟁방지법 등의 법적 문제가 발생함 [00:01:34].
* **올바른 모방 (크로스 오버):** **전연 다른 업종**의 1등 서비스 UI/UX 장점을 참고하여 내 서비스에 녹이는 방식 (예: AI 서비스를 만들 때 카카오톡의 UI나 대형 커뮤니티의 검증된 레이아웃 구조를 차용) [00:02:12]. 
* 나 혼자 개발 연습 및 프로토타입 검증용으로 구현해 보는 것은 법적으로 문제가 없음 [00:02:22].

---

## 3. 혼잡스 전용 'AI 레시피 빌더' 도구 및 프로세스

유튜버가 직접 개발하여 구독자에게 제공하는 플랫폼을 통해 개발 과정을 대폭 단축함.

### 🧱 1단계: 디자인 베이스 및 비주얼 재료 선택 [00:03:36]
* 트래픽이 검증된 대형 사이트들의 구조를 기반으로 한 베이스 레이아웃 선택.
* 브랜드에 맞는 색감, 폰트, 버튼 스타일 등 비주얼 요소를 선택하면 하단의 '냄비(장바구니)'에 UI 재료로 담김.

### ⚙️ 2단계: 필수 기능(Function) 추가 [00:03:56]
* 서비스 운영에 반드시 필요한 핵심 기능들을 모듈식으로 선택.
* *예시 재료:* 회원가입/로그인 기능, 검색창 기능, 게시판 기능 등.

### 📝 3단계: AI 프롬프트 레시피 자동 생성 및 빌드 [00:04:24]
* 담은 재료들을 기반으로 **'레시피 만들기'** 버튼을 누르면 AI가 한 번에 이해할 수 있는 최적화된 프롬프트(레시피)가 자동 생성됨.
* 이 레시피를 복사해 **클로드 코드(Claude Code)**나 **ChatGPT 코덱스**에 붙여넣으면, AI가 단 10분 만에 실제 작동하는 프로토타입 서비스를 구현해 냄 [00:04:32].

---

## 4. 피드백 루프 및 점진적 개선 (Iterative Improvement)

* AI가 10분 만에 뽑아준 초기 UI/UX 샘플을 직접 눌러보며 개선점과 아이디어를 도출 [00:04:42].
* **기능 고도화 기록:** 예시로 초기 이메일 로그인 기능만 있던 샘플에 "구글, 카카오, 네이버 소셜 로그인 기능이 추가되길 원함"과 같은 메모를 보관함에 남겨두고, 다음 턴의 레시피 생성 시 쉽게 결합하여 서비스를 업그레이드해 나감 [00:05:09].

---

## 🚀 실전 1인 창업 적용 프로세스 요약

1. **벤치마킹:** 전혀 다른 업종의 거대 플랫폼 UI 구조를 관찰한다.
2. **모듈 조합:** 검증된 UI 레이아웃 and 필요 기능(로그인, 검색 등)을 컴포넌트 단위로 구조화한다.
3. **AI 전달:** 구조화된 레시피를 클로드 코드에 입력해 빠른 1차 아웃풋을 뽑는다.
4. **메모 & 보완:** 아웃풋의 한계를 체크하고 소셜 로그인 등 고도화 기능을 살을 붙이듯 추가 빌드한다. (EP3에서 실제 구현 예정)
